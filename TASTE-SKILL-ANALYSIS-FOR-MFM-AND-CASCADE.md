## Executive Summary

**Taste Skill** is a collection of specialized directives and configurations designed to enhance how AI agents generate frontend code. It eliminates "AI slop" (generic, uninspired UI patterns) and replaces it with premium, high-agency design engineering through systematic configuration dials and anti-pattern directives.

## What is Taste Skill?

### Core Philosophy
**Eliminate "AI slop" — the generic, uninspired, and often incomplete UI patterns typically produced by Large Language Models — and replace them with premium, high-agency design engineering.**

The system acts as a "system prompt extension" that informs the AI's creative and technical decision-making process for frontend code generation.

### Key Components
1. **Configuration Dials** (3 numeric constants 1-10):
   - `DESIGN_VARIANCE`: Controls layout experimentation (centered/clean to asymmetric/modern)
   - `MOTION_INTENSITY`: Controls animation depth (hover to scroll/magnetic)
   - `VISUAL_DENSITY`: Controls information density (spacious to dense dashboards)

2. **Anti-Slop Directives**:
   - Banned fonts (Inter, Arial)
   - Banned colors (pure black #000000)
   - Banned patterns (3-column grids, purple neon gradients)
   - Banned easing (linear)

3. **Skill Set** (9 skills):
   - **taste-skill**: Default all-rounder for premium frontend output
   - **soft-skill**: Polished, calm UI with premium motion
   - **minimalist-skill**: Editorial, crisp product interfaces
   - **brutalist-skill**: Industrial brutalist UI (BETA)
   - **redesign-skill**: Auditing and upgrading existing codebases
   - **output-skill**: Full-output enforcement (anti-laziness)
   - **stitch-skill**: Google Stitch design system compatibility
   - **gpt-taste**: GSAP-driven Awwwards-level UI
   - **image-to-code-skill**: Image-first pipeline

### Technical Specifications

- **Language**: Markdown-based SKILL.md files

## Analysis for MFM Corporation

### PROS

#### 1. **Dashboard Design Enhancement**
- **Benefit**: Premium design standards for mfm-corp.cc.cd dashboard
- **Impact**: Transforms generic dashboard into premium, high-agency interface
- **Fit**: MFM's dashboard-first approach needs premium design

#### 2. **Visual Density Control**
- **Benefit**: `VISUAL_DENSITY` dial optimizes for data-heavy dashboards
- **Impact**: Better information architecture for agent monitoring
- **Fit**: MFM has 42 agents requiring dashboard monitoring

#### 3. **Anti-Slop Directives**
- **Benefit**: Eliminates generic AI-generated UI patterns
- **Impact**: Professional, consistent design across all interfaces
- **Fit**: MFM needs enterprise-grade presentation

#### 4. **Motion Physics Standards**
- **Benefit**: Spring physics instead of linear easing
- **Impact**: Premium, polished user experience
- **Fit**: Enhances user perception of MFM's capabilities

#### 5. **Redesign Capability**
- **Benefit**: `redesign-skill` for auditing and upgrading existing codebases
- **Impact**: Can upgrade current dashboard to premium standards
- **Fit**: MFM's existing dashboard needs upgrade

#### 6. **Mobile Optimization**
- **Benefit**: Viewport stability rules prevent layout shifting
- **Impact**: Better mobile experience for CEO Remy
- **Fit**: MFM has mobile-optimized interface requirement

#### 7. **Configuration Flexibility**
- **Benefit**: Three dials allow tuning without code changes
- **Impact**: Easy adjustment of design personality
- **Fit**: MFM needs flexible design evolution

#### 8. **Multi-Skill Support**
- **Benefit**: Different skills for different contexts (minimalist, brutalist, soft)
- **Impact**: Context-appropriate design for different sections
- **Fit**: MFM has diverse dashboard sections (agent monitoring, financial, etc.)

#### 9. **Output Enforcement**
- **Benefit**: `output-skill` prevents code truncation and placeholders
- **Impact**: Complete, functional code generation
- **Fit**: MFM needs reliable code generation for dashboard updates

#### 10. **Design System Compatibility**
- **Benefit**: `stitch-skill` for Google Stitch design system
- **Impact**: Enterprise design system integration
- **Fit**: MFM may adopt enterprise design systems

### CONS

#### 1. **Frontend-Focused**
- **Issue**: Primarily designed for frontend code generation
- **Impact**: Limited benefit for backend agent logic
- **Mitigation**: Use for dashboard and interface design only

#### 2. **Learning Curve**
- **Issue**: 9 skills with different use cases
- **Impact**: Requires training for CEO Remy and developers
- **Mitigation**: Start with taste-skill (default) and expand gradually

#### 3. **Design Subjectivity**
- **Issue**: "Premium" design is subjective
- **Impact**: May not align with MFM's brand preferences
- **Mitigation**: Tune configuration dials to match brand

#### 4. **CSS Framework Assumptions**
- **Issue**: Assumes Tailwind CSS and modern CSS features
- **Impact**: May not fit existing CSS architecture
- **Mitigation**: MFM dashboard uses React/Vite with Tailwind (good fit)

#### 5. **Animation Overhead**
- **Issue**: High `MOTION_INTENSITY` may add performance overhead
- **Impact**: Slower dashboard performance
- **Mitigation**: Keep `MOTION_INTENSITY` moderate (4-6)

#### 6. **No Direct Agent Integration**
- **Issue**: Designed for UI code, not agent workflows
- **Impact**: No direct benefit to agent logic
- **Mitigation**: Use for dashboard design, not agent development

#### 7. **Brutalist Skill Beta Status**
- **Issue**: `brutalist-skill` marked as BETA
- **Impact**: May have bugs or incomplete features
- **Mitigation**: Avoid until stable
